OOP

OO imposes discipline about function pointers. (where functions are defined) Functions are object methods that can be inherited (polymorphism)

Data and behavior in a single location (properties and methods)

FP

FP imposes discipline upon assignment. (do not mutate state)

Functional are "first class". We can treat functions like any other data type. They can be stored in arrays, passed around as function parameters, assigned to variables, etc.

OOP Example

var Flock = function(n) {
  this.seagulls = n;
};

Flock.prototype.conjoin = function(other) {
  this.seagulls += other.seagulls;
  return this;
};

Flock.prototype.breed = function(other) {
  this.seagulls = this.seagulls * other.seagulls;
  return this;
};

var flock_a = new Flock(4);
var flock_b = new Flock(2);
var flock_c = new Flock(0);

var result = flock_a // 4
.conjoin(flock_c) // 4
.breed(flock_b) // 8
.conjoin(flock_a.breed(flock_b)) // (16 + (8*2 = 16))
.seagulls; // 32

Side Tangent

associative property - grouping of addends doesn't matter
add(add(x, y), z) === add(x, add(y, z));

commutative - order of addends doesn't matter
add(x, y) === add(y, x);

identity - adding zero doesn't do anything
add(x, 0) === x;

distributive - you can multiply a sum by multiplying each
addend separately and then add the products.
multiply(x, add(y,z)) === add(multiply(x, y), multiply(x, z));

Function Composition

  • Build new functions from other functions
  • Help build generic programs w/o data arguments

( f . g )( x )   =   f ( g ( x ) )

_.compose( f , g ) ( x )   ===   f ( g ( x ) )

_.compose(reverse,toUpper)

var reversedUpperCase   =   _.compose(reverse,toUpper);

reversedUpperCase('hello');   ===   OLLEH

What's the problem with UnderScore?

The Data comes first.

_.map(list,fn)

_.reduce(list,fn)

_.pluck(list,prop)

Function Composition with UnderScore

var lastindexOfTwo = function(arrays) {
  return _.map(arrays,function(array) {
    return _.lastIndexOf(array,2);
  });
};

lastindexOfTwo([[1,2,3,2],[4,5,6,2,5]]);
// [3,3]

_.compose can't save you because collection functions like _.each, _.map, _.reduce all take the collection first

Enter Ramda

A practical functional library that makes it easy to create functional pipelines (functions that are partially applied)

// UnderScore:
var lastindexOfTwo = function(arrays) {
  return _.map(arrays,function(array) {
    return _.lastIndexOf(array,2);
  });
};
// Ramda:
var lastindexOfTwo = R.map(R.lastIndexOf(2));

lastindexOfTwo([[1,2,3,2],[4,5,6,2,5]]);
// [3,3]
